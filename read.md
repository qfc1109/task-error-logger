# Task Error Logger

`task-error-logger` 是一个用于维护项目任务日志和问题记录的技能。它帮助 Codex 在项目根目录的 `task_plan.md` 中记录任务进展、问题分析、配置变更、代码变更、回滚方式和验证结果。

## 适用场景

- 更新当天任务进度。
- 记录 bug、错误堆栈、阻塞点和责任方。
- 记录配置或代码变更，以及后续回滚方法。
- 保存实际执行过的验证命令和结果。
- 维护面向后续工程师可读的工作日志。

## 仓库结构

```text
.
├── agents/
│   └── openai.yaml
├── task-error-logger.md
└── read.md
```

## 文件说明

- `task-error-logger.md`：技能主体，包含触发说明、操作规则、日志模板、状态标记和安全检查。
- `agents/openai.yaml`：OpenAI agent 入口配置，定义显示名称、简介和默认提示词。
- `read.md`：仓库说明文档。

## 使用方式

在需要维护任务日志或问题记录时，调用：

```text
$task-error-logger
```

也可以直接描述需求，例如：

```text
更新 task_plan.md，记录今天修复的登录异常、修改文件、回滚方式和验证命令。
```

技能会优先读取已有的 `task_plan.md`，保留旧内容，并把当天记录追加到对应日期区域。没有实际执行过的验证命令，不会写成已验证结果。

## 记录原则

- 使用 UTF-8 读写日志文件。
- 不覆盖历史记录。
- 不猜测原因；缺少证据时标记为待确认。
- 每条代码变更都要写明文件、修改内容、影响和回滚方式。
- 只有实际运行过的命令才能作为验证证据。

## 维护建议

- 修改技能主体时，优先更新 `task-error-logger.md`。
- 如果默认调用文案变化，同步更新 `agents/openai.yaml`。
- 提交前检查 Markdown 渲染是否正常，尤其是中文标题、状态标记和表格。
